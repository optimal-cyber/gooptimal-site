Optimal Platform is a self-hosted CNAPP. It applies DISA STIG and CIS baselines to your container images before they ship. It scans containers, cloud configuration, runtime workloads, secrets, and SBOM continuously. It emits compliance evidence as a product feature. One SKU. No tiering, no asterisks.

Phase 01 / Harden

Baselines applied before runtime.

Hardening that doesn't lock you into someone else's distro. Optimal applies DISA STIG and CIS profiles to your existing container images. When findings appear, Optimal opens fix PRs against your Git with concrete patches.

Harden your own images. No distro lock-in.

// Harden capabilities

01
DISA STIG benchmark coverage.
Profiles applied to base images and customer images.
02
CIS profile coverage.
Container, Kubernetes, and cloud-config profiles.
03
Fix-PR generation.
GitHub and GitLab. Approval gates respect existing CODEOWNERS.
04
Dockerfile patch synthesis.
Concrete patches, not "consider hardening." Reviewer reads a diff.
05
Helm chart hardening recommendations.
securityContext, capabilities, resource limits, network policy.
06
Hardened-image inventory.
What's hardened, when, against which baseline, by whom.
Phase 02 / Verify

Continuous scanning, continuous truth.

The full CNAPP loop. Containers, cloud configuration, Kubernetes posture, secrets, malware, SBOM. An attack-path graph surfaces toxic combinations in the hour. Drift gets detected at runtime.

Drift surfaces as a finding, not a surprise.

// Verify capabilities

01
Container vulnerability scanning.
Image layers, OS packages, language deps, base-image inheritance.
02
Cloud configuration scanning.
AWS, GCP, Azure. CSPM with policy as code.
03
Kubernetes posture.
RBAC, admission control, network policy, workload identity.
04
Secrets detection.
Code, images, configmaps, env vars. False-positive triage built in.
05
Malware scanning.
Static and behavioural detection in images and running workloads.
06
SBOM generation.
SPDX and CycloneDX. Per-build, per-image, queryable.
07
Attack-path graph.
Toxic combinations: critical CVE × internet-exposed × admin IAM.
08
Drift detection.
Production diverges from hardened baseline; Optimal raises a finding.
09
eBPF runtime monitoring.
Process, network, file. Kernel-level visibility into workload behaviour.
Phase 03 / Prove

Evidence emits as a product feature.

Compliance evidence comes out of operational state, not screenshots. Auditor reads a live feed. POA&M lifecycle tracked through resolution. Shared responsibility matrix drawn explicitly so the customer's compliance team knows where the boundary sits.

Audit time stops being a fire drill.

// Prove capabilities

01
SOC 2 evidence emitter.
CC and additional criteria, mapped to operational state.
02
PCI DSS v4.0.1 evidence emitter.
Requirements 1, 2, 6, 10, 11 covered against your container footprint.
03
HIPAA Security Rule evidence emitter.
Administrative, Physical, Technical Safeguards, with PHI scoping.
04
ISO 27001:2022 evidence emitter.
Annex A controls aligned to Optimal's scanning and hardening output.
05
Shared responsibility matrix.
Explicit boundary: what Optimal does, what the customer does.
06
Incident notification procedure.
Pre-canned, customer-editable, integrated with the finding pipeline.
07
Auditor read-only console.
Scoped account, live feed, no screenshots to package.
08
POA&M lifecycle.
Finding through fix PR through resolution. Tracked, not just opened.
